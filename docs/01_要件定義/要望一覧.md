# 受講生管理システム - 要望一覧

## 1. 機能要望

### 1.1 受講生管理

| 要望ID | 要望内容 | 対応機能 | 優先度 |
|--------|----------|----------|--------|
| REQ-001 | 受講生の情報（氏名、メールアドレス、電話番号、住所、生年月日、性別）を管理したい | STU-001〜007 | 高 |
| REQ-002 | 生年月日から年齢を自動算出してほしい | STU-005 | 高 |
| REQ-003 | 受講生の状態（仮登録、ヒアリング前、ヒアリング後、受講中、修了、退会）を管理したい | STU-006 | 高 |
| REQ-004 | 受講生がどこから申し込んだか（申込経路）を記録したい | STU-001 | 高 |
| REQ-005 | 外部チャットシステム（Discord/Slack）のユーザー名を管理したい | EXT-002, EXT-003 | 中 |

## 2. 非機能要望

### 2.1 性能・可用性

| 要望ID | 要望内容 | 対応 | 優先度 |
|--------|----------|------|--------|
| NFR-001 | 24時間365日稼働してほしい | 可用性要件 | 高 |
| NFR-002 | 同時利用者数10人程度を想定 | 性能要件 | 高 |
| NFR-003 | 受講生データ1,000〜10,000人を想定 | 性能要件 | 高 |

### 2.2 セキュリティ

| 要望ID | 要望内容 | 対応 | 優先度 |
|--------|----------|------|--------|
| NFR-010 | 自前認証（ユーザー名 + パスワード）+ JWTで認証したい | 認証方式 | 高 |
| NFR-011 | 多要素認証は不要 | 認証方式 | - |
| NFR-012 | パスワードポリシーは業界標準（NIST準拠）に沿う | パスワードポリシー | 高 |

### 2.3 インフラ・運用

| 要望ID | 要望内容 | 対応 | 優先度 |
|--------|----------|------|--------|
| NFR-020 | Railway でフロント・バック・DBを一括管理したい | ホスティング環境 | 高 |
| NFR-021 | GitHub Actions で CI/CD を行いたい | CI/CD | 高 |
| NFR-022 | バックアップは1日1回、7日間保持 | バックアップ | 高 |
| NFR-023 | 監査ログは3年間保持 | ログ管理 | 高 |

### 2.4 対応環境

| 要望ID | 要望内容 | 対応 | 優先度 |
|--------|----------|------|--------|
| NFR-030 | Chrome、Firefox、Safari、Edgeに対応したい | 対応ブラウザ | 高 |
| NFR-031 | Phase 1ではレスポンシブ対応不要（デスクトップのみ） | レスポンシブ | - |
